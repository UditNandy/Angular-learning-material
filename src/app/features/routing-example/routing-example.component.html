Routing

  • In angular routing helps us to navigate from one part of the application to another part.
  • To use routing we have build in router module @angular/router
  • To achieve routing we need to create a routes array in app.module.ts and in the imports we need to add RouterModule.forRoot(routes)
  • Another way is to create a routing module and performing all the above actions their and then import it to the main module.
  • To make a not found page we need to specify the path as ** which means any and we need to place it in the button because if we place it anywhere above then at any point ** will get matched and not found page will appear even if we have the path.
  • We can navigate to different pages using href but here the each time href executes the page reloads which means a new request will be sent to the serve and it will return a new page.
  • We can use routerLink in place of href to navigate to different pages through html
  • Router Link is a directive that binds the HTML element to a Route. We can also pass additional parameters using it.
  • Router Link Active is a directive for adding or removing classes to a HTML element that is bound to that RouterLink Using this we can toggle css classes for active links based on current router state.It is used to highlight which route is currently active.
  • There are two types of route paths
    • Absolute: In this case the path is directly appended to the root url. If we use / infront of any router link then its a absolute path.
    • Relative : In this the path is appended to the currently active route, but only if the route is present inside the currently active route. Here we donot use / at the starting.
  • Route Parameters are the dynamic part of routes whose value changes.
  • Example : /route/:id
  • We can access the params from ActivateRoutes.snapshot.params or ActivateRoutes.snapshot.paramMap or ActivateRoutes.paramas observable
  • Child Route is a route inside a route.
  • We can create a child route by simply adding a children array inside the any route object of Routes array and remeber don't give the component name there instead add component names to child routes only.

Route Guards

  • They add validation before navigating to a route or navigating away from a route.
  • There are five types of route guards in angular :
    • CanActivate : It is used to allow or restrict a user from navigating a route.
      • To implement this we need to create a auth guard service and implement CanActivate Interface
      • We have to call canActivate method which must return boolean or {{ "Observable<'boolean'> " }} or {{ "Promise<'boolean'>" }}
      • Assign the canActivate property to the route in the routes array in array format.
      • In Angular 15 + we can add a CanActivate in the routes array inplace of placeing the route gurad and the function must return the required types.
    • CanActivateChild : It is used to when we have to restrict the user to visit the parent route but not the child routes. To achieve this we can use canActivateChild in the routes array and we need to implement the canActivateChild interface in the route guard.
    • CanDeactivate : It is used to allow or restricting a user from navigating away from a route. Example : Your changes might not be saved if you go away.
    • Resolve : These guards delay the activation ofthe route untill some tasks are completed. Example : If weneed to fetch something from backend before we navigate to a route.
    • canLoad : They prevents the loading of lazy loaded modules. Example : when we don't want unauthorized used to even see the source code of the module.
