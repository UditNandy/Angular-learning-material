View Encapsulation

  • It is a behaviour in angular where component's CSS styles are encapsulated within the component's view and donot effect the rest of the application.
  • It helps in leaking css styles out of a component and affecting other parts of the application.
  • Generally css styles are global scoped.
  • In angular there are 3 types of view encapsulation.
    • Emulated (default) : Here angular attaches a unique attribute to all the component's elements and the css classes are applied only on those unique attributes.Each component has its own unique attribute which is same througout the component.
    • None : It means no view encapsulation will happen. We can use the css classes present in the parent's css style sheet.To achieve this we need to add encapsulation:ViewEncapsulation.None in component metadata.
    • Shadow DOM : Here if any global css is present then in that case we can use shadow dom to remove the component from the affect of global css.
