Template Reference Variable

  • It is a variable which stores the reference to a DOM element, component or a directive on which it is used.
  • We can create it by prefixing a variable with #.
  • Example : <input #myInput />. Here #myInput is the template reference variable.

@ViewChild

  • View Child decorator is used to query and get reference of a DOM element in the component.
  • If there are multiple decorators with same template reference variable, it will point to the first matching reference variable.
  • Syntax : @ViewChild('templateChild',{{ "{static:true}" }}) {{ "childVariable!: any;" }}. Here templateChild is the template reference variable.
  • Static Determines when the query has to be resolved.
    • true : When the view is initialised(after the first change detection cycle) for the first time.
    • false : Resolved after every change detection cycle.
  • We can use it with any element, component or directive.

@ViewChildren

  • It is same as @ViewChild but @ViewChild only returns the first element which matches the reference. Where as @ViewChildren returns a list of all the elements that matches the reference.

Content Projection

  • < ng-content > is a placeholder element in Angular that is used to project content into a component from the outside.
  • When you use it within a component's template, any content that we include between the opening and closing tags of the component selector will be inserted into the placeholder.
  • If we have multiple ng-content in a same html file then the content passed will be replaced by the last ng-content.
  • It we want to project different content in different ng-contents in that case we can use the select attribute.
