Angular JSON

  • Note : When we use ng new we create a angular workspace which contains a default application my-app.
  • $schema :
    • It is used for defining definition and structure for rest of the current JSON file.
    • It is used to validate the structure of angular file.
    • It ensures that the configuration file is valid and accoding to Angular CLI's schema.
  • version : Used to maintain the version of angular.json schema.
  • newProjectRoot :
    • It contains the default root folder where new applications/libraries will be placed.
    • It only effects the future applications/libraries.
  • projects :
    • It contains the list of projects created in the new workspace.
    • It is an object where key is project name and value is project configuration.
    • Against each project we will have the following properties.
    • projectType : It states whether the current project is an application or library.
    • schematics :
      • These are templates or blueprints that help automate tasks like creating components, services, modules, etc. They are built-in or custom templates used to generate code, configure projects, or modify project files in an Angular workspace.
      • These configurations specify the options that control how Angular CLI commands, such as ng generate or ng add, behave when generating or modifying components, services, or other elements.
      • Angular CLI has a set of built-in schematics for generating different project parts, such as components, services, directives, modules, etc. You can customize the default behavior of these schematics by defining options in angular.json.
      • skipTest true : means ng g c command will not generate the spec files.
      • In short these can be used to control the angular cli command or add some default behaviour to the components generated using angular cli like style sheet : scss , or change detection statergy onPush, etc
    • root :
      • Base path of the app.
      • Empty means root of the workspace.
      • It its blank means the project is present in root of workspace.
    • sourceRoot : Means the root of the src folder.
    • prefix : Used to specify prefix of component selectors. In our case it is app. We can set it to something else.
    • architect : It contains the build, serve, test, i18, etc configurations.
      • build
        • builder:{{ "@angular-devkit/build-angular:browser" }} : Defines how the build process should be done. The value here is a npm package which contains various builders. The builder is responsible for the build process of the project.
        • options: Contains outputPath, index, main, pollyfills, tsconfig, assets, styles, scripts, etc. These are the entry points for the application. When the build will be generated these entry points will be used to generate the final bundle.
        • configurations : It contains production, development configurations. These production and development configurations contains instructions for fileReplacements,budgets,hashing,etc.
  • defaultProject : It contains the defaut project of the angular app.
