Change Detection

  • Angular Runs a change detection cycle whenever any state gets changed or any event is triggered.
  • This is really important as it helps us to keep our template and class file in sync with each other.
  • Change detection can happen either by two ways :
    • default : Here change detection will run for all the changes and it might hamper the performance. Example : If change detection cycle runs on the app component then all the child component's change detection cycle will also run.
    • OnPush : By using OnPush statergy change detection for the component will only work under the following circumstances :
      • If the input properties are changed and that also reference of the input properties got changed.
      • Any change occured in the child.
      • Subscribing to an observable.
      • Or manually using change detection methods.
      Methods of changeDetectorRef :
      • Detach : It will detach the change detection
      • Reattach : It will reattach the change detection
      • DetectChange : Here we will be able to detect changes even if the change detection is detached.
  • Zone js : It refers to a context that helps manage and track asynchronous operations. They are used to automatically trigger change detection whenever any asyn event occur.
  • Zone.js : It is a library that provides execution contexts(zones) which are responsible for handling change detection by events from async operations.
  • NgZone : It is a service which provides functionalities to manually handle zones, add something in the zone or remove something out of it.
