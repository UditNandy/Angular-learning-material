  • constructor vs ngOnInit :
    Constructor is a typescript feature. Whenever a class is instantiated its constructor is called. It is not ideal for complex logics as in this part the angular component is not instantiated properly.It is generally used for dependency injection in components.
    ngOnInit is a lifecycle hook. It is a perfect place to write some logics after initialisation of the component.

  • Different types of compilers in Angular : JIT and AOT
  • JIT :
    • Here the html an the typescript files are converted into javascript file in the browser durring the runtime.
    • Instead of compiling the code all at once it interpretes the component when they are called in browser.
  • AOT :
    • Here the html and typescript code is converted to javascript code at first which is downloaded by the browser and executed.
    • Here the code is already compiled before it is executed
    • Due to the before hand compilation the rendering becomes faster.
