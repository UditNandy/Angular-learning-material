Prototype

  • In javascript each and every object has a prototype property.
  • It is a place where we put properties or methods which we want the instances of that method to inherit.
  • Prototype is always associated with the constructor function. __proto__ is always associated with the instances of that function constructor.
  • In every object or arrays , etc you can always see a __proto__ property. It is the prototype.
  • Thats the place from where we get the .length(), .keys() and all the other methods that we commonly use in our daily programing.
  • Take an example of declaring a constructor function for tempaling a user blueprint.
  • The user function will have name, dob, and calcAge method.
  • Now if we use calcAge method as a part of constructor funtion then remember for all the instances of the constructor function we will get calcAge and some repetative task will take place.
  • Here prototype will come into picture.
  • Instead of keeping calcAge as a part of constructor function, we can declare it outside and attach it to Person.prototype.
  • In this way all the object created using the constructor function will be able to access calcAge but this time calcAge is a part of their prototype and not their actual body.
  • udit.__proto__ === Person.prototype

Inheritance

  • It is when one object is based on another object, i.e when one object get access to the properties and methods of another objects.
  • In javascript you can achieve inheritance using prototype.
  • When we attach a property or method to the prototype property of that object, those properties and methods can be inherited by the instantiated object.

Prototype Chaining

  • It is a feature in javascript which allows objects to inherit the properties and methods from other objects by creating a chain of prototypes.
  • When you try to access a property or method on an object, javascript will first look for that property directly on the object. If it doesnot fint it, it will look for the property in its prototype and if if doesnot find it there as well it will again look for the property in the prototype's prototype and this will go on till the prototype is null.
