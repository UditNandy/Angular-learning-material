Pipes

  • Pipes in Angular are used to transform or format data before displaying it in the view.
  • It takes the data as input and it formats or transforms that data before displaying it in the template.

Custom Pipes

  • While creating a custom pipe the PipeTransform Interface needs to be implemented.
  • PipeTransform Interface will provide us a transform method in which we will get two parameters value and the args we can do the operations and return the new value.The value parameter will take the data on which we are implementing the pipe.The second parameter has to be sent by : .
  • Percentage Pipe example: {{ 0.11 | customPercentage : 1 }}
  • Async pipe example: {{ obs$ | async }}

Pure Pipe

  • It is a pipe which gets called whenever there is a pure change of input value.
  • By default every pipe is a pure pipe.
  • If a pipe is being used on a primitive type like string, boolean, number, etc and if the value of that primitive type changes, it is considered as pure change.
  • If a pipe is being used on a reference type and the reference changes that it will also considered as pure change.

Inpure Change

  • If a pipe is being used on a reference type and the reference does not changes only it's value changes then it will be considered as an impure change.
  • A pipe is known as impure if it gets executed irrespective of its pureness in every change detection cycle.
  • To make a pipe impure we can add pure:false in pipe metadata.
  • We should avoid using impure pipes because it will get executed for all the change detection cycle.
  • For sorting and filtering we should not use pipes because in case of pure pipe changes will not reflect properly if any value gets added and in case of impure pipe it will get executed in every change detection cycle.
