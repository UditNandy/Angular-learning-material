Template From

  • When we submit a normal form in HTML, the whole page gets reload by making a HTTP request to the server.
  • As we are creating a SPA, we can say that it will restart the entire Angular app.
  • To avoid this we need Angular to stop this default behaviour.
  • This can be achieved by Template or reactive forms.
  • To work with forms in Angular we need FormsModule.
  • We can use ngSubmit in place of onSubmit to submit the form.
  • We need to assign a template reference to the form and template reference should be ngForm.
    Syntax : #myForm = "ngForm"
  • ngModel directive will tell angular that if it is used if input element inside a template form then that input element is a control.
  • We can use dirty property of a form for CanDeactive where we can restrict the user from navigating away from route if the form is dirty and not submitted.
  • Note : Incase of radio buttons always keep the name same for different options.
  • To check if a form is valid or not we can use form.vaild.
  • A form group is a collection of form controls. In template form we can achieve it using ngModelGroup.
  • To access the properties of a form group we need to store that form group in a templete reference variable.
    Syntax : #address="ngModelGroup"
  • If we want to add some value from ts then we can make use of setValue but we have to pass all the controls here.
  • If we need to pass only the control whose value we want to set we can make use of patchValue.

Advantages

  • Less coading is required in class file.
  • Easy to understand and better for small forms.

Disadvantages

  • Adding controls dynamically durring runtime is a challange.
  • Not suitable for complex forms.
