Compilation Process

  • High level language : The code we write using programing languages like c, java, javascript. These are human readable languages.
  • Machine Code : The high level programs are not understood by computer's so they are converted into 0's and 1's which is known as machine code.
  • Conversion of high level language to machine code can be done by compilation or interpretation.
  • Compilation : Here the entire source code is converted to machine code at once, which is then written into a protable file (like binary file) which can be executed by any operating system or browser at any time.
  • In compilation steps involved are :
    • Conversion of source code(high level language) to machine code and writing it in a file.
    • Execution of machine code to run the program.
    • Execution can happen any time way after its compilation.
  • Interpretation : Here we have an interpreter which goes through the code line by line and executes it durring run time.
  • Interpreted languages are much slower then compiled languages.
  • Modern javascript uses a combination of compilation and interpretation.
  • JIT(Just in time compilation) :
    • Here the javascript code is converted to machine code and gets executed right away.
    • Here there is no portable file but the execution happens immediatly after the compilation.

Javascript compilation

  • Parsing :As soon as we run a javascript program the javascript engine parse it line by line and checks for syntax errors.
  • If any syntax error is found then the engine throws an error or else if everything is right the parser produces a data structure known as Abstract Syntax Tree (AST).
  • AST; This tree is used for generating the machine code.
  • The machine code generated gets executed right away.
