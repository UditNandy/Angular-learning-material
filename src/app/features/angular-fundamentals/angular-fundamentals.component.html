Angular Fundamentals

Angular files and folders

  • node_modules :
    • All the 3rd party libraries gets stored here on which our Angular app might depend on.
    • All the 3rd party libraries we download from npm gets stored in this node module folder.
    • This is purely for development process and we don't deploy this folder in production server or checkin this folder in the git repo.
  • package.json :
    • This is a standard configuration file which every node project has.
    • It has the meta data of the project like name, version, scripts, dependencis, devDependencies.
    • In dependencies we will have the list of all packages on which the project is directly dependend, devDependencies is the list of all packages which are required in the time of development of the project.
  • .editorconfig : It is file which is used to set rules related to coding standards. In real world project many different people work in a same project and all may have different approach of writing code.
  • .gitignore : here we can specify the files and folders that we want to ignore.
  • angular.json: It contains akk the configuration of our angular project, like what are the stylesheets the app is going to use and where to find it, the main entry point, etc.
  • package.lock.json:
    • It keeps a record of exact version of every installed dependency, including their sub dependencies and versions.
    • It makes sure that the same set of dependencies are present accross different environment.
  • tsconfig.json : It basically contains a bunch of settings for our typescript compiler. Typescript compiler is goinf to loot at these settings and based on these settings the compiler will change the typescript file into javascript file.
  • src : Here we contains the source code of our application.

Bootstrapping angular app works

  • Bootstrapping is the process of loading our angular app.
  • index.html is the first page in angular app.
  • The index.html file doesnot have any javascript file attached to it but we can see an app-root tag in the body.
  • app-root:
    • To run our angular app we need to type a command ng-serve. This commnad actually builds our app but doesn't save it in the disk rather it saves it in the memory. This command also watches our project and if we make any changes then it will recompile and update the build file.
    • ng-serve bundles the application and injects it to the index.html file.
    • To view this build we need to run an angular cli command called ng build.
    • After building the angular app we will get a dist folder and inside the folder we will have a index.html file too there we will see that inside the body tag there is app-root and a new script tag is imported.
    • In angular webpack bundles the angular application.
    • Once the angular cli has these bundles then angular cli injects this bundles in the index.html file.
  • So by the time the index.html file is loaded, Angular core modules and thirdparty modules are also loaded.
  • Now agular needs to locate the main entry point of the application.
  • The angular cli searches for the entry point in angular.json where in main the entry point is mentioned.
  • In the main.ts file we need to import platformBrowserDynamic module which is responsible for loading the angular app in the browser.
  • Angular app can be bootstrapped in many ways and in many platforms like mobile devices with ionic or native scripts
  • In the main.ts we have to bootstrap the AppModule class.
  • Inside app module we have a bootstrap array in which we have to specify some components.
  • The components specified in the AppModule bootstrap array will be loaded as soon as the AppModule loads.

Different js files found in build

  • vendor.js :
    • This is generated durring the build process of an Angular app.
    • It contains all the external libraries on which our project relias like Rxjs, Angular Material, etc.
  • polyfills.js :
    • It plays a vital role in ensuring that our angular application runs smoothly across different browsers and versions.
    • It provides modern JS features to old browsers which does not support them directly.
  • main.js:
    • It is the entry point of our angular application.
    • It contains the bootstrap logic and initialise and launches our Angular application.
    • When the browser loads main.js, it triggers the Angular Framwork to start which in turn bootstraps to app-root.
  • runtime.js :
    • It provides the runtime environment necessary to run our Angular app.
    • It contains the core angular runtime code, which helps angular to achieve change-detection, dendency injection ,routing,etc.

proxy.config.js

  • It is an object where keys can be used to find the endpoints starting with that key.
  • target : This is the target backend server.
  • secure : A flag to decide whether to verify SSL certificates when using HTTPS in the target. Tells the proxy to validate / not validate the backend's SSL certificate.
  • changeOrigin : Tells the proxy server to modify the Host header in the outgoing request to match the target server. Some backend servers block requests if the Host header does not match their domain. In the request headers we can find :authority: which is the current host where the app is being served.So this will be changed to the target host.
  • logLevel : Controls the amount of log output the proxy gives in your terminal. Values are : debug, info, warn, error, silent.
  • pathRewrite : Allows you to modify the request path before it is sent to the backend.
  • We can add it in ng serve command ng serve --config proxy.config.json
